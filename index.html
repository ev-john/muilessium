Overview

Muilessium is the UI framework created for the fast development of the websites. Simple websites which constitute a significant part of the internet. No SPA. No jQuery. No heavy plugins. Just a number of components and helpers - all that you need to develop a website. It has a REM-based sizes system for non-designers and a simple scheme of usage: Sketch → Construct → Paint → Use.

Grid

Muilessium uses a grid with 12 columns based on flexbox (and has a legacy version based on floats, but only in custom builds). Rows and columns have classes .mui-row and .mui-column and columns can have options -s* for a small screen, -m* for a medium screen and -l* for a large screen. Default breakpoints are 400, 600 and 1200px. Its behavior is similar to the grid from flexboxgrid.com, but it uses RSCSS-style class names. See src/less/components/grid-flex.less for more information.

Recommended viewport

Muilessium uses Hammer.js, and it is recommended to use the following meta in your pages:

    <meta name='viewport' content='user-scalable=no, width=device-width, initial-scale=1, maximum-scale=1'>

Typography

Typography in Muilessium is fluid, and it is based on default system fonts and _bigger-* and _smaller-* helpers. Note: these helpers have different behavior on different screen sizes to avoid troubles with readability.

Events

Muilessium has a system of internal events. It exports that system as window.Muilessium.Events. The main methods are:

  • addEvent(name)
  • addEventListener(name, callback, executeIfAlreadyFired=false)
  • fireEvent(name)

There are six default events:

  1. resize-window-height
  2. resize-window-width
  3. scroll-start
  4. scroll-end
  5. app-initialized
  6. images-loaded

Colors

A number of classes for painting from src/less/colors. There are four types of classes:

  • -color-number - text color for the element
  • -color-number-bg - background for the element
  • -f-color-number - text color for the :hover and :focus states of the element
  • -f-color-number-bg - background color for the :hover and :focus states

Components

Muilessium has a number of components listed below. Every component has a markup shown in example and this markup should be used as is, because some initializations of components are based on their markups. All components with public methods are stored in window.Muilessium.components. If the main element of such component has an ID, the component can be obtained using Muilessium.get(componentName, id).

  • getValue() - returns input value
